# ESP32_CAM_WiFiManager_Stream

Учебный скетч для ESP32-CAM AI Thinker.

Он показывает более удобную схему работы с Wi-Fi:

```text
первый запуск -> GUI WiFiManager -> подключение к роутеру -> IP в Serial Monitor -> веб-страница с кнопкой stream
```

## Что делает скетч

- поднимает временную точку доступа `ESP32-CAM-SETUP`, если Wi-Fi еще не настроен;
- открывает портал настройки Wi-Fi по адресу `http://192.168.4.1`;
- сохраняет SSID и пароль во внутренней памяти ESP32;
- после подключения печатает IP в Serial Monitor;
- запускает веб-страницу камеры на `/`;
- запускает MJPEG-видеопоток на `/stream`;
- пробует включить mDNS-имя `http://esp32cam.local/`.

## Библиотеки

В Arduino IDE должна быть установлена библиотека:

```text
WiFiManager by tzapu
```

Также нужен установленный ESP32 core для Arduino.

## Параметры платы

Рекомендуемые настройки Arduino IDE:

```text
Board: AI Thinker ESP32-CAM
Upload Speed: 115200
Serial Monitor: 115200
```

## Первый запуск

1. Прошей ESP32-CAM.
2. Отключи `GPIO0` от `GND`.
3. Перезагрузи плату.
4. Подключись к Wi-Fi сети:

```text
ESP32-CAM-SETUP
```

Пароль:

```text
12345678
```

5. Открой:

```text
http://192.168.4.1
```

6. Введи данные своего роутера.
7. После подключения смотри IP в Serial Monitor.

## Адреса после подключения

Главная страница:

```text
http://IP_КАМЕРЫ/
```

Видеопоток:

```text
http://IP_КАМЕРЫ/stream
```

mDNS-вариант, если поддерживается компьютером или телефоном:

```text
http://esp32cam.local/
```

## Сброс сохраненного Wi-Fi

В скетче есть строка:

```cpp
// wm.resetSettings();
```

Для сброса Wi-Fi настроек временно сделай так:

```cpp
wm.resetSettings();
```

Прошей и запусти плату один раз. Потом снова закомментируй строку.

## Почему поток вынесен на /stream

В исходном простом варианте поток часто висит прямо на `/`. Здесь сделано удобнее:

- `/` — обычная HTML-страница с IP и кнопкой;
- `/stream` — сам MJPEG-поток.

Так новичку проще понять, какой IP получила камера и какую кнопку нужно нажать.
